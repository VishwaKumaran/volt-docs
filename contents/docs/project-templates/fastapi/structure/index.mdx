A typical Volt-generated FastAPI project follows a modular and scalable structure.

<FileTree>
  <Folder name="app" label="Source code">
    <Folder name="core" label="Core config, security settings" />
    <Folder name="models" label="Database models" />
    <Folder name="routers" label="API route handlers" />
    <Folder name="schemas" label="Pydantic schemas" />
    <Folder name="services" label="Business logic layer" />
    <File name="main.py" label="App entry point" />
  </Folder>
  <Folder name="tests" label="Pytest test suite" />
  <File name=".env" label="Environment variables" />
  <File name="pyproject.toml" label="Dependency & tool configuration" />
  <File name="README.md" />
</FileTree>

## Description

-   **app/**: Contains all source code.
-   **routers/**: Routes are split into modules (e.g., `users.py`, `items.py`) and aggregated in a router.
-   **core/**: Central configuration using Pydantic Settings.
-   **tests/**: Mirror the `app` structure for unit and integration testing.
-   **pyproject.toml**: The single source of truth for dependencies and tool settings (Ruff, Pytest, etc.).
